# 追記セクション：C型（運用再現）／再計算型（分析用）の役割分離（確定）

## 背景（何が起きたか）
90D検証において、**再計算型（分析用）に CAP10 と 場除外（運用制約）を適用**すると、
- モデル性能の評価（conf分布・ROI）
- 運用上の制約による件数減（CAP10・除外）
が混在し、結果の解釈が困難になる。

特に「**C型の件数＝再計算型の件数**」という先入観があると、Sが見えている日でも90D側で件数が0になる現象を“不具合”と誤認しやすい。

## 用語定義（確定）

### ① C型（運用再現 / Cフェーズ正本）
**目的**：その日の実運用の意思決定を忠実に再現する  
**処理順**：
1) official_features × fan × policy で **conf 算出**  
2) **S/A 判定**（S: conf ≥ 0.87 / A: 0.60–0.86 / 見送り < 0.60）  
3) **場除外（運用都合）**  
4) **CAP10（最大10R・最低保証なし）**  
5) 投票内容（tickets_long）を確定（= C正本）

### ② 再計算型（分析用 / 90D検証）
**目的**：モデル構造の性能（ROI・安定性）を評価する  
**処理順**：
1) 90D分の official_features × fan × policy で **conf を再計算**  
2) **conf ≥ 0.87 を S相当として抽出**（※分析用のためS/Aラベル化しない）  
3) 期間集計（件数・ROI 等）

## 確定ルール（結論）

- 再計算型（分析用）：**CAP10 ❌ / 場除外 ❌（原則）**  
  ※入れる場合は「別軸の比較（what-if）」として明示し、baseline評価と混ぜない
- C型（運用再現）：**CAP10 ⭕ / 場除外 ⭕**（運用制約として適用）

> 1行まとめ：**再計算型は「性能を見る」だけの世界。制約は載せない。CAP10と場除外は「判断を再現する」C型だけに適用する。**
